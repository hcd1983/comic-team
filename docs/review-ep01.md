# 第一話創作流程檢討報告

> 作品：《社恐女孩的內心小劇場》第一話「早安地獄」
> 日期：2026-04-03
> 參與檢討：編劇顧問、分鏡顧問、美術總監、製作人

---

## 一、整體摘要

團隊首次完整走過「編劇討論 → 分鏡生成 → AI 作畫」流程。故事方向和分鏡節奏基本到位，但 **AI 生圖品質** 存在嚴重問題（角色不一致、文字錯誤、畫風不統一），**編劇討論機制** 和 **資料結構** 也有明顯改善空間。

---

## 二、各階段檢討

### 階段 A：編劇討論

| 問題 | 說明 |
|------|------|
| 編劇之間沒有真正互動 | 第二輪各自提案，本質是「平行報告」而非「交叉碰撞」，浪費了團隊協作最有價值的即時激盪 |
| 提案缺乏差異化約束 | 阿龍在被要求「各提不同方向」後仍提勇者題材，顯示缺少明確的禁區與品質門檻 |
| 大綱產出不夠完整 | story-outline.json 只有基本角色和起承轉合，缺乏具體場景、對話節奏、情緒轉折點、視覺演出提示，不足以支撐分鏡工作 |
| 導演角色過於被動 | 只做了「排序」，沒有主動引導討論方向、整合不同構想的優點 |

**改善建議：**
1. 改為序列式提案：A 先提 → B 讀 A 後提不同方向 → C 讀 A、B 後再提，確保差異化
2. 討論前由導演先給出限制條件（如：禁止異世界題材、需要日常系）
3. 選定方向後加入「三人共同深化」環節，借用所有人的腦力完善被選中的方案
4. 定義 story-outline.json 的最低規格（見第三節）
5. 加入「笑點測試」環節：至少三個具體笑點場景在定稿前驗證

### 階段 B：分鏡生成

| 問題 | 說明 |
|------|------|
| 描述偏敘事性 | 「辦公室走廊，同事打招呼」對人類夠用，但對 AI 生圖資訊密度不足 |
| 缺乏角色設定錨點 | 每格獨立描述角色，沒有共用的角色外觀設定，導致服裝、場景跨格不一致 |
| 分鏡到 prompt 的轉譯斷層 | 分鏡描述是「創作意圖」，prompt 需要「視覺指令」，中間缺少結構化轉換層 |
| 使用者審核不夠細緻 | 使用者只能整包確認或整包重來，缺乏逐格微調機制 |

**改善建議：**
1. 建立結構化 prompt 模板，每格強制包含：構圖比例、場景描述、每個角色的位置與姿態、表情精確描述、畫風關鍵字
2. 角色設定表獨立管理，每格 prompt 引用同一份設定
3. 分鏡產出後提供逐格審核與微調機制

### 階段 C：作畫生成

| 問題 | 嚴重度 | 說明 |
|------|--------|------|
| 中文文字不可控 | **P0** | 第2格中文變英文、第3格「社交能力」→「社金能力」、第4格「嗯」→ 日文「うん」 |
| 角色一致性崩壞 | **P0** | 第1格 OL 套裝 → 第4格休閒服，辦公室 → 學校走廊，四格敘事邏輯斷裂 |
| 畫風不統一 | **P1** | 第1格寫實日系 → 第2格半 Q 版 → 第4格不同風格寫實，像四個不同繪師的拼貼 |
| 缺乏品質閘門 | **P1** | 生成後沒有任何自動化檢查就直接輸出 |

**改善建議：**
1. **文字完全後製化（P0）**：prompt 加入 `no text, no speech bubbles, no dialogue`，所有文字在後製階段用標準漫畫字型疊加
2. **角色設定圖先行（P0）**：正式生成前先用 AI 產出角色三視圖（正面/側面/表情差分），作為後續每格的視覺參考
3. **統一風格 prompt 前綴（P1）**：建立固定的畫風關鍵字組合，每格必帶
4. **品質閘門（P1）**：每格生成後用 vision model 做圖文一致性比對 + 跨格一致性檢查
5. **考慮替代方案（P2）**：測試支援角色參考的模型（SD XL + IP-Adapter、Midjourney --cref），或採用「AI 生底圖 + 人工後製」混合流程

### prompt 模板

每格生成的 prompt 應遵循以下結構：

```
[風格前綴] 4-koma manga style, flat cel shading, bold black outlines, soft pastel background, no text, no speech bubbles, no dialogue.
[構圖] 中景 / 特寫 / 遠景，畫面比例 1:1
[場景] {引用 scenes 欄位的描述}
[角色] {引用 characters 欄位的 appearance + outfit + 指定 expression}，位於畫面{位置}，{具體姿態}
[情緒] {氛圍關鍵字}
```

## 四、優先改善排序

| 優先序 | 項目 | 負責 | 預期效果 |
|--------|------|------|---------|
| **P0** | 文字完全後製化，prompt 禁止生成文字 | 美術/工程 | 徹底解決文字亂碼問題 |
| **P0** | 新增角色設定表結構 + 角色設定圖先行 | 編劇/美術 | 解決角色跨格不一致的根本原因 |
| **P0** | 建立結構化 prompt 模板 | 分鏡/工程 | 降低 AI 自由發揮空間，提升可控性 |
| **P1** | 編劇討論改為序列式 + 導演主動引導 | 編劇/導演 | 避免重複提案、提升討論品質 |
| **P1** | 作畫品質閘門（vision model 比對） | 工程 | 自動攔截品質不合格的圖片 |
| **P2** | 分鏡逐格審核與微調機制 | 工程/UX | 提升使用者控制精度 |
| **P2** | 測試替代生圖模型（SD XL + IP-Adapter） | 美術/工程 | 評估角色一致性更好的方案 |
| **P3** | 作畫平行化生成 | 工程 | 縮短等待時間 |
| **P3** | 流程指標追蹤（耗時、重試次數等） | 工程 | 量化流程效率，持續優化依據 |

## 五、核心結論

> **AI 生圖應定位為「高效草稿產生器」而非「成品輸出機」。**

本次最大的教訓：

1. **文字渲染不可靠** — 目前沒有任何 AI 圖像模型能可靠地生成中文文字，必須後製處理
2. **角色一致性需要視覺錨點** — 純文字描述無法讓逐格獨立生成的圖維持同一角色，需要角色參考圖機制
3. **編劇討論需要真正的碰撞** — 平行各說各話不是討論，序列式互評才能讓點子進化
4. **資料結構是品質的基礎** — story-outline.json 的完整度直接決定下游所有階段的品質上限

下一次創作前，優先完成 P0 項目，預期可大幅提升產出品質。
