# 替代生圖模型評估計劃

> 基於第一話檢討：Gemini 2.5 Flash Image 在角色一致性和中文文字方面表現不佳

## 評估框架

| 維度 | 權重 | 說明 |
|------|------|------|
| 角色一致性 | 30% | 跨格角色外觀（髮型、服裝、體型）維持能力 |
| 畫風穩定性 | 25% | 四格之間線條風格、上色方式的一致程度 |
| 構圖可控性 | 20% | 對鏡頭角度、人物位置、姿態指令的遵從度 |
| API 可用性 | 15% | 穩定性、速度、rate limit、整合難度 |
| 成本 | 10% | 每張圖生成的價格 |

**不評估中文文字能力**：所有模型的文字生成都不可靠，已決定全部後製處理。

## 候選模型

### 1. Gemini Imagen 4.0
- **類型**：Google 專用圖片生成模型
- **優點**：同生態系，整合簡單
- **關注**：與 Gemini Flash Image 的畫質差異

### 2. Stable Diffusion XL + IP-Adapter
- **類型**：開源模型，本地部署
- **優點**：IP-Adapter 可餵入角色參考圖，大幅提升一致性
- **缺點**：需要本地 GPU、部署成本高
- **關注**：角色一致性是否真的比純 API 好

### 3. Midjourney (--cref + --sref)
- **類型**：商用 API
- **優點**：`--cref` 鎖定角色、`--sref` 鎖定風格，業界一致性最佳
- **缺點**：API 存取限制、價格較高
- **關注**：四格漫畫風格是否支援

### 4. DALL-E 3 / GPT-4o Image
- **類型**：OpenAI API
- **優點**：構圖可控性強、prompt 遵從度高
- **缺點**：角色一致性無專門機制
- **關注**：與 Gemini 的比較

### 5. nano-banana-pro-preview
- **類型**：Google 實驗模型
- **優點**：已在可用模型列表中
- **關注**：與 Gemini Flash Image 的差異

## 評估實驗設計

### 測試素材
使用第一話「早安地獄」的分鏡，同一份 storyboard.json + promptTemplate。

### 測試步驟
1. 用同一份 prompt 在各模型生成四格圖片
2. 由團隊成員（或 vision model）對五個維度評分（1-5）
3. 計算加權總分
4. 記錄每張圖的生成時間和成本

## 後續行動

- 評估完成後，根據結果決定是否更換模型
- 若更換，需修改 `scripts/gemini-draw.mjs` 或新增對應的 MCP server
