# 編劇討論

啟動漫畫編劇討論流程。多位編劇 AI 輪流發言，使用者做最終決策。

## 流程

1. 詢問使用者故事主題或大綱（若未提供）
1.5. 讀取素材包（若存在）：
   - 若 `output/{slug}/{version}/material-pack.json` 存在，讀取並在討論前呈現摘要（topInsights、avoidList、suggestedAngles）
   - 若不存在，提示使用者可先執行 `/research` 搜集素材，或直接跳過進入討論
   - 素材包的內容會作為 context 傳給每位編劇，讓他們在討論時引用
1.8. 導演開場：根據主題和素材包，給出本次討論的限制條件：
   - 禁止的題材類型（如：避免老套的異世界/勇者）
   - 目標受眾
   - 必須包含的元素（若有）
   - 參考素材包的 `avoidList` 和 `suggestedAngles`

2. **第一階段：序列式提案**（每位編劇提出不同方向）
   - 呼叫 `scriptwriter-a`（阿龍），傳入主題 + 限制條件 + 素材包
   - 呼叫 `scriptwriter-b`（小雪），傳入主題 + 限制條件 + 素材包 + **阿龍的提案**（必須提出不同方向）
   - 呼叫 `scriptwriter-c`（老謝），傳入主題 + 限制條件 + 素材包 + **阿龍和小雪的提案**（必須提出第三個方向）
   - 每位編劇**必須提出與前面編劇不同的方向**

3. 提案結束後，以下列格式呈現：

```
═══ 提案階段 ═══

【阿龍】提案 A：{標題}
（提案內容）

【小雪】提案 B：{標題}
（提案內容）

【老謝】提案 C：{標題}
（提案內容）

───────────────────────────────────
請選擇：選定方向（A/B/C）/ 混搭元素 / 讀者投票 / 全部重來
```

4. 使用者選定方向後，進入 **第二階段：三人共同深化**
   - 三位編劇依序針對被選中的方案補充細節：
     - 阿龍：補強動作場面和節奏設計
     - 小雪：深化角色心理和對話設計
     - 老謝：完善結構、伏筆和世界觀

```
═══ 共同深化 ═══

【阿龍】節奏與場面補強
（補充內容）

【小雪】角色與對話深化
（補充內容）

【老謝】結構與伏筆完善
（補充內容）

───────────────────────────────────
請選擇：確定方向 / 繼續深化 / 讀者投票 / 補充意見
```

5. **笑點測試**（搞笑類作品）：
   - 定稿前列出至少 3 個具體笑點場景描述
   - 使用者評估每個笑點是否有效
   - 無效的笑點需要重新設計

5.5. **四格劇本品質檢查**（四格漫畫時必做）：
   確認劇本是否符合四格鐵則，逐項檢查：
   - [ ] 場景是否統一（最多一個場景或同場景兩角度）？
   - [ ] 時間是否連續（同一時間段，無大跳躍）？
   - [ ] 是否只有一個核心笑點（非多事件流水帳）？
   - [ ] 第三格是否有明確的反轉（靠角色反應而非事件）？
   - [ ] 角色行為是否有動機（非為搞笑而搞笑）？
   - [ ] 是否避免了追逐戲或動作過場？
   若有不符合項，要求編劇修改後才能進入下一步。

6. 使用者選擇：
   - **確定方向** → 請使用者總結決定的故事方向，進入第 7 步
   - **繼續深化** → 回到第 4 步
   - **讀者投票** → 依照 `/vote` skill 流程，根據作品 TA 自動推薦讀者投票，結果供參考
   - **補充意見** → 使用者提供額外方向，帶入下一輪
   - **換方向** → 重新回到提案階段

7. 使用者確定方向後，將結果整理成以下 JSON 並儲存至 `output/{slug}/{version}/story-outline.json`：

```json
{
  "title": "故事標題",
  "theme": "使用者原始主題",
  "synopsis": "100 字以內的故事摘要",
  "characters": [
    {
      "id": "角色 ID（英文，如 jing）",
      "name": "角色名",
      "personality": "性格描述與行為特徵",
      "appearance": "外觀細節（髮型、髮色、五官、膚色、體型）",
      "outfit": "服裝描述（款式、顏色、配件）",
      "expressions": {
        "default": "預設表情的精確描述",
        "表情名稱": "該表情的精確描述"
      }
    }
  ],
  "scenes": [
    {
      "id": "場景 ID（英文，如 office-hallway）",
      "description": "場景視覺描述（色調、光源方向、主要物件配置、地板材質、天花板特徵）"
    }
  ],
  "plotPoints": [
    {
      "beat": "起 / 承 / 轉 / 合",
      "scene": "場景 ID（引用 scenes）",
      "characters": ["角色 ID（引用 characters）"],
      "event": "具體事件描述",
      "emotion_target": "預期讀者情緒反應"
    }
  ],
  "tone": "搞笑 / 熱血 / 溫馨 / 懸疑",
  "style": "畫風描述（如：日系漫畫風格，可愛線條）"
}
```

**注意**：`characters` 的 `appearance` 和 `outfit` 必須足夠詳細，讓 AI 生圖時能精確重現角色外觀。`scenes` 的 `description` 必須包含色調和光源，確保跨格場景一致。

8. 告知使用者討論結論與檔案路徑。
