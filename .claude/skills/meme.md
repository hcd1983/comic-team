# 梗圖 / 兩格對比生成

用現有角色 IP 生成梗圖或兩格對比圖。不需要完整劇本，只需要一個梗或一個對比。

## 格式選擇

1. **單格梗圖**：一張圖 + 大字文案
2. **兩格對比**：上下或左右對比（before/after、理想/現實、期待/結果）
3. **反應圖**：角色對某個情境的表情反應

## 流程

### 步驟 1：選擇角色與主題

1. 確認使用的角色 IP（若有現有角色設定圖則引用）
2. 詢問主題方向，或由編劇提案：
   - 使用者指定主題 → 直接進入步驟 2
   - 使用者沒指定 → 呼叫一位編劇（阿龍），用「梗圖模式」提 3 個梗：
     - 每個梗用一句話說完
     - 必須不需要背景知識
     - 必須光看圖就懂（不依賴文字也能有 7 成效果）

### 步驟 2：確認梗的內容

呈現格式：
```
═══ 梗圖企劃 ═══

格式：兩格對比 / 單格梗圖 / 反應圖
梗：{一句話描述}
上格/左格：{畫面描述}
下格/右格：{畫面描述}
文案：{圖上的文字，大字，5 字以內}

秒懂測試：不看文字光看圖能懂嗎？ ✅ / ❌
```

使用者確認後進入步驟 3。

### 步驟 3：生圖

- 呼叫 `gemini_draw` MCP tool 生成圖片
- 若有角色設定圖，傳入 `referenceImages` 維持角色一致性
- 兩格對比：分別生成兩張圖
- 單格/反應圖：生成一張
- 風格遵循角色 IP 的畫風設定

### 步驟 4：加文字（可選）

- 使用 FFmpeg drawtext 在圖上燒文字
- 大字、粗體、白字黑邊
- 位置：上方或下方

### 步驟 5：輸出

- 儲存至 `output/{slug}/{version}/memes/meme_{N}.png`
- 若是兩格對比，額外產出合併版（上下拼接）

## 編劇梗圖模式規則

當編劇被呼叫時，傳入以下限制：
- 你在想「梗圖」不是在寫劇本
- 一句話說完，不要展開
- 笑點在畫面，不在文字
- 用角色的動物特質當笑點
- 不要隱喻、不要暗諷、不要需要想的梗
- 格式：「{上格畫面} vs {下格畫面}」或「{一個畫面} + {一句話}」
