# 品質閘門

每格漫畫圖片生成後的自動品質檢查。

## 檢查項目

### 1. 圖文一致性檢查
使用 vision model（Claude 自身）讀取生成的圖片，比對：
- 角色數量是否正確
- 角色姿態是否符合分鏡描述
- 場景元素是否吻合
- 鏡頭角度是否正確

### 2. 跨格一致性檢查
比對當前格與已生成格的：
- 角色髮型/髮色是否一致
- 服裝款式/顏色是否一致
- 角色體型/頭身比是否一致
- 相同場景的背景元素是否一致

### 3. 文字殘留檢查
檢查圖片中是否出現：
- 不應有的文字或字母
- 不應有的對話氣泡
- 亂碼或殘留符號

## 結果分級

| 等級 | 條件 | 處理 |
|------|------|------|
| **PASS** | 三項檢查全部通過 | 繼續下一格 |
| **WARN** | 有小問題但不影響閱讀（如背景細節略有差異） | 提示但繼續 |
| **FAIL** | 角色嚴重不一致、出現文字、場景完全錯誤 | 自動重新生成 |

## 重新生成策略

FAIL 時的處理：
1. 分析失敗原因
2. 修改 prompt（加強失敗點的描述精確度）
3. 重新呼叫 `gemini_draw`
4. 最多重試 2 次
5. 仍 FAIL 則標記為 `MANUAL_REVIEW`，供使用者手動處理

## 檢查流程

```
生成圖片
    ↓
讀取圖片（vision）
    ↓
├── 圖文一致性 → PASS / WARN / FAIL
├── 跨格一致性 → PASS / WARN / FAIL
└── 文字殘留   → PASS / FAIL
    ↓
綜合判定（取最嚴重等級）
    ↓
├── PASS → 繼續
├── WARN → 提示 + 繼續
└── FAIL → 修改 prompt + 重新生成（最多 2 次）
```

## 結果輸出格式

```
【第 N 格品質檢查】
  圖文一致性：PASS ✓
  跨格一致性：WARN ⚠ （髮色略有偏差）
  文字殘留：  PASS ✓
  綜合判定：  WARN — 繼續（已記錄）
```
