# 讀者投票

啟動虛擬讀者投票模式。根據作品 TA 自動推薦讀者組成投票團，對爭議選項進行投票。

## 觸發時機

- 編劇討論出現分歧（提案階段 / 深化階段）
- 分鏡審核有多版本難以抉擇
- 使用者主動要求 `/vote`

## 流程

### 步驟 1：確認投票問題

向使用者確認：
- **投票問題**：要決定什麼？
- **選項列表**：A / B / C...（每個選項附一句簡述）

若在 `/discuss` 或 `/storyboard` 流程中觸發，自動從上下文提取問題和選項。

### 步驟 2：TA 匹配與讀者推薦

讀取 `config/ta-presets.json` 和 `config/virtual-readers.json`。

**自動推薦流程**：

1. **判斷作品 TA**：根據以下資訊判斷（優先順序）
   - `story-outline.json` 中的 `targetAudience` 欄位（若已存在）
   - 討論過程中導演設定的目標受眾
   - 使用者指定的 TA 預設（如「少年熱血漫」）
   - 若以上皆無，詢問使用者

2. **匹配 TA 預設**：從 `ta-presets.json` 找到最接近的預設，取得推薦讀者清單

3. **組成投票團**（5-7 位）：
   - 取預設推薦的前 5 位作為核心
   - 確保多樣性：至少 2 種性別、至少 2 個年齡層
   - 若多樣性不足，從預設推薦的候補中替換

4. **呈現投票團名單**，讓使用者確認或調整：
   ```
   本次投票團（依 TA「少年熱血漫」自動推薦）：
   小炎(16M/高中生) | 阿肥(21M/大學生) | 虎爺(29M/健身教練)
   蜜蜂(31F/行銷) | 泡芙(17F/高中生) | 拓海(42M/餐廳老闆)
   阿南(38M/司機)

   確認此組合 / 換人 / 改用其他 TA 預設
   ```

### 步驟 3：執行投票

依序呼叫每位被選中的讀者 agent（`reader-01` ~ `reader-20`），傳入：
- 投票問題
- 選項列表與簡述
- 作品的一句話概要（不傳完整素材包，控制 token）

每位讀者回覆格式：`選 X — 「一句話理由（30 字內）」`

**重要**：5-7 位讀者可平行呼叫，不需序列等待。

### 步驟 4：彙整結果

```
═══ 讀者投票結果 ═══

【投票問題】{問題描述}

【小炎】選 A — 「{理由}」
【阿肥】選 A — 「{理由}」
【蜜蜂】選 B — 「{理由}」
【泡芙】選 B — 「{理由}」
【虎爺】選 A — 「{理由}」
【拓海】選 A — 「{理由}」
【阿南】選 B — 「{理由}」

📊 統計：A 票 x4 / B 票 x3
──────────────────────────────
此為讀者觀點參考，最終決定權在您手上。
```

### 步驟 5：使用者決策

呈現選項：
- **採納多數** — 按票數最高的方向繼續
- **自行決定** — 無視投票結果，使用者自選
- **追問讀者** — 針對特定讀者追問理由（此時該讀者可回覆 50 字內補充）
- **換一批讀者重投** — 更換投票團組成重新投票

## 注意事項

- 投票結果**僅供參考**，絕不自動執行
- 讀者 agent 使用 `haiku` model，控制成本
- 若投票結果出現完全平票，呈現平票事實，由使用者裁決
- 同一議題最多投票 2 次（避免無限循環）
